## 一、方法论对齐

### 1.1 BLM 方法论原文要求

> 宏观环境分析主要采用 PEST 分析方法，从宏观的角度洞察和判断国家层面的
> 政治、经济、社会、技术等方面的变化与发展趋势，进而分析这些趋势将会给
> 行业带来什么样的影响与变化。

> 目的是帮助企业依托宏观势能（如新政策的出台）来调整业务方向，
> 加速新业务的发展或退出停滞且未来会消失的业务。

### 1.2 "看宏观"最终要回答的问题

来自方法论的明确要求：

1. **市场正在发生的重大变化有哪些？**
2. **产业政策有哪些重大改变？对企业影响是怎样的？**
3. **产业格局的变化带给我们的影响、机遇和挑战有哪些？**
4. **整体市场空间有多大？增长率是多少？利润将发生什么样的变化？**
5. **新技术的发展趋势及变化是怎样的？本企业可参与的市场空间有多大？**

这些问题直接衔接"看机会"的 Addressable Market 输出。

## 二、PEST 四维度详细定义

### 2.1 总览

| 维度 | 英文 | 核心问题 | 比喻 |
|------|------|---------|------|
| **P** | Political | 政策/法规环境对我的经营有什么约束或推动？ | 游戏规则 |
| **E** | Economic | 经济环境影响消费者付费能力和我的成本结构吗？ | 大气候 |
| **S** | Society | 社会/人口/文化变化如何改变需求？ | 用户画像的底层变化 |
| **T** | Technology | 技术趋势创造了什么新机会或替代风险？ | 赛道的改变 |

每个维度都要输出：**现状描述 → 变化趋势 → 对电信行业的影响 → 对本企业的具体影响**

### 2.2 P — Political（政治/政策/法规）

> 对组织经营活动具有实际与潜在影响的政治力量和有关的法律法规等因素。

| 分析要素 | 电信行业的具体内容 | 数据来源 |
|---------|-----------------|---------|
| **电信监管政策** | 频谱分配/拍卖规则、网络共享政策、互联互通费率、号码携带规则、网络中立性 | BNetzA / BEREC / EU 法规 |
| **网络安全立法** | NIS2 指令实施、关键基础设施保护要求、供应商安全审查（如华为限制） | BSI / EU 网络安全局 |
| **数据保护法规** | GDPR 执法趋势、跨境数据流动规则、电子隐私条例 | 数据保护机构 |
| **国家数字战略** | 千兆战略、光纤覆盖目标、5G 覆盖要求、数字主权政策 | 联邦政府战略文件 |
| **产业补贴与扶持** | EU 复苏基金中的数字化投资、农村宽带补贴、5G 专网政策 | EU / 联邦经济部 |
| **税收与经营法规** | 公司税率变化、数字税、频谱使用费、环保合规要求 | 联邦财政部 |
| **竞争法与并购** | 市场支配力审查、并购审批规则、反垄断调查 | 联邦卡特尔局 |
| **国际政治因素** | 中美科技竞争对供应链的影响、地缘政治对跨国运营的影响 | 政治分析 / 媒体 |

### 2.3 E — Economic（经济环境）

> 宏观经济环境 + 微观消费环境，直接决定企业目前及未来的市场大小。

| 分析要素 | 电信行业的具体内容 | 数据来源 |
|---------|-----------------|---------|
| **宏观经济指标** | GDP 增长率、GDP 总量、经济周期阶段 | IMF / Destatis |
| **通胀与物价** | CPI、能源价格指数、工资增长——影响运营成本和定价空间 | ECB / Destatis |
| **就业与收入** | 失业率、可支配收入、消费者信心指数——影响消费者付费意愿 | Eurostat |
| **利率与融资** | ECB 基准利率、信贷环境——影响网络投资的融资成本和回报要求 | ECB |
| **能源成本** | 电力价格、天然气价格——运营商是用电大户，直接影响 OPEX | 能源监管机构 |
| **汇率**（跨国运营商）| 欧元/英镑/美元汇率——影响跨国运营商的合并报表 | ECB |
| **消费结构变化** | 通信支出占家庭支出的比例变化、数字服务支出增长 | Eurostat 家庭调查 |
| **企业 IT 支出** | 企业数字化投资预算变化——影响 B2B 市场空间 | Gartner / IDC |

### 2.4 S — Society（社会环境）

> 居民受教育程度、人口结构、文化水平、价值观念等。

**这是之前设计中缺失的维度。** 对电信行业的影响：

| 分析要素 | 电信行业的具体内容 | 数据来源 |
|---------|-----------------|---------|
| **人口结构** | 老龄化趋势、移民人口增长——影响数字服务采纳率和用户结构 | Destatis 人口统计 |
| **数字素养** | 互联网普及率、智能手机渗透率、数字技能水平——影响新服务的推广难度 | Eurostat 数字经济指标 |
| **消费行为变化** | 流媒体替代传统 TV、远程办公常态化、在线购物渗透——影响流量需求和业务结构 | 行业调研 |
| **环保与可持续** | ESG 意识上升、碳减排要求、绿色消费偏好——影响网络建设方式和品牌形象 | ESG 报告 / EU 绿色协议 |
| **城乡差异** | 城市 vs 农村的数字化差距——影响网络部署优先级和政府补贴方向 | BNetzA 覆盖报告 |
| **隐私意识** | 消费者对数据隐私的关注度提升——影响数据驱动业务的发展空间 | 消费者调研 |
| **劳动力市场** | IT/电信人才供给、薪资水平——影响运营商的人才招聘和成本 | 行业薪资报告 |

### 2.5 T — Technology（技术环境）

> 不仅包括发明，还包括与企业市场有关的新技术、新工艺、新材料的出现
> 和发展趋势以及应用背景。

| 分析要素 | 电信行业的具体内容 | 数据来源 |
|---------|-----------------|---------|
| **网络技术演进** | 5G SA/NSA 进展、5G-Advanced、6G 研究方向、Wi-Fi 7 | 3GPP / 标准组织 |
| **固网技术** | FTTH/FTTB 部署技术、DOCSIS 4.0、XGS-PON、50G-PON | 行业技术报告 |
| **AI 与自动化** | AI 对网络运维的优化、AI 对客服的替代、生成式 AI 创造的新需求 | 技术媒体 / 分析师 |
| **云与边缘计算** | 边缘节点部署、MEC（多接入边缘计算）、云网融合 | 行业报告 |
| **开放网络** | OpenRAN 成熟度、供应商生态、成本效益——1&1 是先驱 | O-RAN Alliance / 分析师 |
| **卫星互联网** | Starlink / Kuiper 等低轨卫星对地面网络的补充或替代 | FCC / 技术媒体 |
| **安全技术** | 量子计算对加密的威胁、零信任架构、网络安全即服务 | 技术媒体 / 行业报告 |
| **国家科技投入** | 政府对 6G 研发的投入、AI 国家战略、半导体自主化 | 政府科技规划 |
| **技术商品化速度** | 从实验室到商用的速度、专利保护——影响先发优势的持续时间 | 专利数据库 / 行业分析 |

## 三、分析输出的层次

每个 PEST 维度都要按四层输出：

```
第一层：现状（Facts）
  → 当前的政策/经济/社会/技术环境是什么状态

第二层：变化趋势（Trend）
  → 正在发生什么变化，方向是什么

第三层：对电信行业的影响（Industry Impact）
  → 这个变化对整个电信行业意味着什么

第四层：对本企业的具体影响（Company Impact）
  → 对我来说是机会还是威胁，程度如何
```

### 数据模型更新

```python
@dataclass
class PESTFactor:
    """一个 PEST 因素"""
    dimension: str                # "P" / "E" / "S" / "T"
    dimension_name: str           # "Political" / "Economic" / "Society" / "Technology"
    factor_name: str              # "千兆战略" / "GDP 增长" / "老龄化" / "5G SA"
    
    # 四层分析
    current_status: str           # 现状描述
    trend: str                    # 变化趋势
    trend_direction: str          # "improving" / "worsening" / "stable" / "uncertain"
    industry_impact: str          # 对电信行业的影响
    company_impact: str           # 对本企业的具体影响
    
    # 判断
    impact_type: str              # "opportunity" / "threat" / "neutral" / "both"
    severity: str                 # "high" / "medium" / "low"
    time_horizon: str             # "short_term" / "medium_term" / "long_term"
    predictability: str           # "predictable" / "uncertain" / "volatile"
    
    # 溯源
    evidence: list[str]           # 支撑数据
    data_source: str              # 数据来源

@dataclass
class PESTAnalysis:
    """看宏观的完整 PEST 输出"""
    # 按维度组织
    political_factors: list[PESTFactor]
    economic_factors: list[PESTFactor]
    society_factors: list[PESTFactor]
    technology_factors: list[PESTFactor]
    
    # 方法论要求的关键输出
    market_size_impact: str       # 整体市场空间及增长率的判断
    profit_trend: str             # 利润将发生什么变化
    policy_opportunities: list[str]  # 产业政策带来的机遇
    policy_threats: list[str]     # 产业政策带来的挑战
    tech_addressable_market: str  # 新技术趋势下可参与的市场空间
    
    # 综合天气判断
    overall_weather: str          # "sunny" / "partly_cloudy" / "cloudy" / "stormy"
    weather_explanation: str      # 天气判断的理由
    key_message: str              # 一句话总结
```

## 五、与"看机会"的衔接

方法论要求"看宏观"输出的几个关键问题，需要直接传递给"看机会"：

```
看宏观输出                                看机会接收
───────────                              ──────────
market_size_impact                  →    整体市场空间判断的背景
  "德国电信市场 €13.4B, +1.2%"

profit_trend                        →    利润变化对机会评估的影响
  "行业利润率承压但触底回升"

policy_opportunities                →    政策驱动的机会点
  ["千兆战略补贴", "EU 复苏基金"]          每个对应一个 OpportunityItem

tech_addressable_market             →    技术趋势创造的新市场空间
  "5G 企业专网市场 €800M (2027)"          对应 OpportunityItem 的 addressable_market
```

## 七、替换之前的设计

本文档替换之前"五看分析框架详细设计"中"四、看宏观"的全部内容。

关键变更：
1. **四维度 → PEST 四维度**：用标准 PEST 框架替代之前自定义的四维度
2. **新增 Society 维度**：人口结构、数字素养、消费行为、ESG、城乡差异
3. **Technology 维度扩展**：不只是行业技术，还包括国家科技投入、技术商品化速度
4. **每个因素四层分析**：现状 → 趋势 → 行业影响 → 对本企业的影响
5. **与"看机会"的显式衔接**：市场空间、利润趋势、政策机遇、技术市场直接传递
6. **PPT 仪表盘更新**：四象限标签改为 P/E/S/T，每个因素标注机会/威胁/中性
